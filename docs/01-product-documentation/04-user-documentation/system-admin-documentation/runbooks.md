# Runbooks

Copy-pasteable operational procedures. Author each as the process becomes real.

## Planned runbooks
- **Deploy backend** — cut a release and verify health.
- **Rollback backend** — revert to a previous image safely (migration-aware).
- **Release mobile** — submit and promote iOS/Android builds.
- **Update province data** — safely refresh the canonical
  [datasets](../../01-core-specifications/api-system-specifications/data-schemas.md).
- **Replay failed events** — inspect and replay entries in the Modulith event-publication log.
- **Incident response** — triage → mitigate → communicate → postmortem.

> This is the index until the procedures exist; then split into one file per runbook if it grows.
