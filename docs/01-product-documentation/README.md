# 1.1 Product Documentation

Describes **what** VieGo is and how the system behaves. Four categories:

| # | Section | Purpose |
|---|---------|---------|
| 1.1.1 | [Core Specifications](01-core-specifications/) | The **source of truth**: API/system specs + executable (BDD) specs |
| 1.1.2 | [Authored System Documentation](02-authored-system-documentation/) | High-level context not in the spec: architecture (C4, DDD, Modulith) + UX |
| 1.1.3 | [Generated System Artifacts](03-generated-system-artifacts/) | Auto-generated from the core specs |
| 1.1.4 | [User Documentation](04-user-documentation/) | End-user, developer integration, system admin |

**Flow:** Core Specifications (1.1.1) are authored/agreed first; Generated Artifacts (1.1.3)
are produced from them; Authored System Docs (1.1.2) explain the context the specs can't; User
Docs (1.1.4) teach people to use and operate it.
